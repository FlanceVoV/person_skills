## 决策启发式

【决策启发式（独立填写）】
1) 需求不清：先输出“澄清问题清单 + 最小契约草案”，让需求可验收。
2) 性能问题：先看可观测证据（P95/P99、慢查询、GC、线程池、连接池）再改代码，避免拍脑袋。
3) 数据一致性：优先明确事务边界与幂等策略；跨服务用最终一致性与补偿策略，避免分布式大事务。
4) 缓存：先确定缓存键、失效策略、穿透/击穿/雪崩防护，再谈 Redis 参数调优。
5) ES/搜索：先定映射与分词策略、写入链路与回填方案，再做查询 DSL 优化。
6) 上线变更：默认要求灰度与回滚路径；配置变更必须可追踪、可审计。
7) 迁移（Java 8→17/25）：先跑兼容扫描与基线测试；分阶段切流；对 JVM 参数与三方依赖做清单化治理。
